# TRPG 봇 초기 뼈대 정리

## 1) 프로젝트 목표
디스코드 텍스트 명령어(`!명령어`) 기반으로,
플레이어가 계정 생성부터 캐릭터 생성까지 순차적으로 진행하고,
기존 플레이어는 로그인 즉시 게임 시작 상태로 진입하도록 구성했습니다.

---

## 2) 현재 구현된 핵심 흐름

### 시작
- `!TRPG시작`
- `!티알피지시작`
- `!티얼피지시작`

시작 명령어를 입력하면,
- **처음 사용자**: 1단계(아이디/비번 생성)부터 가이드
- **기존 사용자**: 로그인 가이드 출력

### 계정/로그인
1. `!아이디 <아이디>`
2. `!비번 <비밀번호>`
3. `!로그인 <아이디> <비밀번호>`

- 비밀번호는 salt + PBKDF2 해시로 저장됩니다.
- 기존 플레이어가 로그인하면: **"게임 시작됩니다!"** 메시지가 출력됩니다.

### 캐릭터 생성 단계
4. `!닉네임 <닉네임>`
5. `!직업설정 <전사|궁수|힐러|도적>`
6. `!주사위 던지기`
7. `!캐릭터생성` (또는 `!캐릭터 생성`)

각 단계 완료 시 다음 단계 가이드가 메시지로 출력됩니다.

### 캐릭터 조회
- `!캐릭터 조회`

---

## 3) 스탯/직업 설계

### 기본 주사위
- 스피드(speed): 1d6
- 공격력(attack): 1d6
- 방어력(defense): 1d6

### 직업 보정치
- 전사: speed +0 / attack +1 / defense +2
- 궁수: speed +1 / attack +2 / defense +0
- 힐러: speed +1 / attack +0 / defense +1
- 도적: speed +2 / attack +1 / defense +0

최종 스탯 = 기본 주사위 + 직업 보정치

---

## 4) 데이터 저장 구조
기본 루트: `playerdata/`

- `playerdata/auth/`: 계정 정보 JSON
- `playerdata/setup/`: 생성 단계 진행 상태 JSON
- `playerdata/players/`: 최종 캐릭터 JSON

이 폴더는 Git 추적 대상이며 ignore 하지 않습니다.

---

## 5) 코드 구조
- `execute.py`: 실행 엔트리포인트 (`python execute.py`)
- `text.py`: 봇 생성/실행(`create_bot`, `run_bot`)
- `CommandManager.py`: 디스코드 명령어 등록
- `PlayerDesign.py`: 플레이어 도메인 로직/저장/검증

---

## 6) 현재 상태 요약
- 초기 사용자 온보딩 완료
- 계정/로그인 흐름 완료
- 캐릭터 생성 흐름 완료
- 단계별 가이드 메시지 완료
- 기존 플레이어 로그인 시 게임 시작 메시지 완료

다음 단계는 전투/턴/스킬/인벤토리 같은 실제 게임플레이 시스템 확장입니다.
